## Описание
В ходе выполнения данной лабораторной работы были изучены основы взаимодействия JavaScript с HTML-документом через объектную модель DOM. Основные изученные темы:
* **Поиск элементов:** использование методов `getElementById` и `querySelector`.
* **Манипуляция содержимым:** изменение текста через `textContent` и стилей через свойство `style`.
* **Обработка событий:** работа с методом `addEventListener`, обработка кликов и отправки форм.
* **Валидация данных:** проверка пользовательского ввода в формах перед обработкой.
* **Динамическое создание элементов:** использование `createElement` и `appendChild` для обновления интерфейса без перезагрузки страницы.
* **Делегирование событий:** оптимизация обработки событий для динамически созданных объектов.

## Структура проекта
* `index.html` — основная разметка с примерами работы DOM и формами.
* `style.css` — стилизация элементов интерфейса.
* `main.js` — логика работы с поиском элементов, событиями и валидацией.
* `dynamicElements.html` / `dynamicElements.js` — реализация динамического списка пользователей.
* `img/` — папка со скриншотами этапов выполнения работы.

## Сравнение JavaScript DOM и C# WinForms

| Концепция | C# (WinForms) | JavaScript (DOM) |
| :--- | :--- | :--- |
| **Найти элемент** | `this.Controls["name"]` | `document.getElementById("name")` |
| **Изменить текст** | `label1.Text = "..."` | `label.textContent = "..."` |
| **Обработчик** | `button1.Click += Handle` | `btn.addEventListener("click", Handle)` |
| **Создать элемент** | `new Button()` | `document.createElement("button")` |
| **Скрыть элемент** | `Visible = false` | `style.display = "none"` |

**Вывод:** DOM и WinForms решают схожие задачи по управлению интерфейсом, однако DOM является кроссплатформенным решением, работающим в любом браузере.
